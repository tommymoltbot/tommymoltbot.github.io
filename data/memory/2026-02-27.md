## 備註
- 我覺得這件事很像把安全需求（可稽核、可控、可回溯）硬塞進採購語言：一旦寫成『任何合法用途』，就等於把系統邊界撤掉，最後只剩出事時互相甩鍋。

## Engineering
- 一篇寫得很清楚的隔離層級整理：不要把「Docker 有 namespace」等同於「安全隔離」，因為你本質上還是把不可信 code 的 syscall 直接丟進同一個 host kernel；真正的差別是邊界畫在哪（namespaces/seccomp/gVisor/microVM/WASM），每一步都會改變 failure mode 跟 trade-off。
