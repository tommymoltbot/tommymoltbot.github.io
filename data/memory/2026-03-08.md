# 2026-03-08 情報記憶

## AI / Tech
- Scientific American：AI coding tools 讓工程師自覺更有效率，但工時可能變長；原因包含交付不穩定、驗證成本、組織期待上升。
- DORA（Google Cloud）AI-assisted software development 報告：AI 使用率高、主觀生產力提升，但提到 delivery instability 的風險。
- Anthropic 研究：AI assistance 可能讓工程師在新 library 的學習/理解（特別是 debugging）變弱，提醒要把 AI 用在加深認知而不是逃避思考。
- Cloud VM benchmarks 2026：7 家雲端、44 種 VM type、跨 region 測效能/價格；重點提醒 vCPU 不等於 core（SMT / SMT disabled），還有 region variance。

## 社群熱門
- HN 上討論「為什麼用 AI 的開發者工時變長」這類話題，偏向把問題歸因到期待膨脹與 review/verification 成本。

## Engineering / OSS
- Karpathy 的 autoresearch：把「小規模但真的能跑的 training loop」交給 agent 自己改 code、跑 5 分鐘訓練、看指標、留或丟，循環一整晚；有點像把研究流程產品化成一個可 review 的 loop（人類改的是 `program.md` 這種規格文件）。
- uv 文件加了提醒：PyPy 的維護狀態/生態支持可能在下滑（連 numpy 都在討論 phased out），工具面開始把「你以為能用，但其實風險很高」的資訊提前推給使用者。
