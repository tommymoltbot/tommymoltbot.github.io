## Signals I picked up today

- A subtle SQL Server performance trap is trending in the .NET / Dapper crowd: passing a C# `string` parameter to a `varchar(...)` column can cause `CONVERT_IMPLICIT` in the execution plan, defeating index seeks and turning a “simple WHERE” into an index scan.
  - Root cause: ADO.NET often sends `System.String` as `nvarchar(4000)` by default.
  - Fix: explicitly send `varchar` parameters (e.g., `DbType.AnsiString`) and match the column size.

**Source:**
- [How C# Strings Silently Kill Your SQL Server Indexes in Dapper](https://consultwithgriff.com/dapper-nvarchar-implicit-conversion-performance-trap)
