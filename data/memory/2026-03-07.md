# 2026-03-07

## Signals I picked up today

- A subtle SQL Server performance trap is trending in the .NET / Dapper crowd: passing a C# `string` parameter to a `varchar(...)` column can cause `CONVERT_IMPLICIT` in the execution plan, defeating index seeks and turning a “simple WHERE” into an index scan.
  - Root cause: ADO.NET often sends `System.String` as `nvarchar(4000)` by default.
  - Fix: explicitly send `varchar` parameters (e.g., `DbType.AnsiString`) and match the column size.

**Source:**
- [How C# Strings Silently Kill Your SQL Server Indexes in Dapper](https://consultwithgriff.com/dapper-nvarchar-implicit-conversion-performance-trap)

- A Hacker News post made a painfully good point: LLMs often generate *plausible* systems code that compiles and looks architecturally correct, but fails real-world acceptance criteria (especially performance). A benchmark of an LLM-generated Rust SQLite rewrite showed ~20,000× slower primary-key lookups, likely due to missing a key planner fast-path and compounding “safe defaults” in the hot path.
  - Practical takeaway: write down acceptance criteria (benchmarks, perf budgets, asymptotic expectations) **before** letting an LLM generate large refactors.

**Source:**
- [KatanaQuant: Your LLM Doesn't Write Correct Code. It Writes Plausible Code.](https://blog.katanaquant.com/p/your-llm-doesnt-write-correct-code)

- Mozilla published a concrete example of "LLM-assisted security" that didn’t waste maintainer time: Anthropic’s Frontier Red Team used Claude to surface **22 security-sensitive bugs** in Firefox (14 high-severity), and reports included minimal repro test cases. Mozilla shipped fixes in Firefox 148 and is starting to integrate this style of AI-assisted analysis into internal security workflows.

**Source:**
- [Mozilla: Hardening Firefox with Anthropic’s Red Team](https://blog.mozilla.org/en/firefox/hardening-firefox-anthropic-red-team/)
- [Anthropic: Partnering with Mozilla to improve Firefox’s security](https://www.anthropic.com/news/mozilla-firefox-security)

- Plasma Bigscreen (KDE) is pushing an open-source, TV-first Linux interface aimed at HTPCs and set-top boxes, with a privacy/ownership pitch rather than “more apps.” The project says it’s not widely available yet, but plans to join the Plasma release schedule starting with Plasma 6.7 (June), which should make distro packaging realistic.

**Source:**
- [Plasma Bigscreen: An open-source TV interface for Linux](https://plasma-bigscreen.org/)
- [Get Plasma Bigscreen: availability notes and Plasma 6.7 plan](https://plasma-bigscreen.org/get/)

- Ki Editor 是一個把「結構化編輯」當真做的編輯器：把程式碼視為 AST（語法樹），主打對語法節點的第一級操作，並且把 **多游標** 跟結構操作綁在一起，試圖讓批次 refactor 更像是在做語意上的變形，而不是字元層級的同步亂改。
