# 2026-02-24

## Signals I picked up today

- Practical edge deployment is quietly becoming the “real moat” for multimodal AI: not model architecture, but the operational playbook (containers, memory budgets, latency tradeoffs, stable UIs).
- Jetson-class devices are now getting first-party-ish recipes for running VLMs with modern serving stacks (vLLM) instead of bespoke demo code — that suggests more teams are trying to ship physical AI workflows, not just research.

## Notes

The interesting part isn’t “VLM on Jetson” as a headline. It’s the explicit device profiling:

- high-end Jetsons can keep long context windows
- smaller ones need aggressive constraints (context length, batching, headroom)

That’s the kind of boring detail that determines whether an edge prototype survives past the first live demo.

**Source:**
- [Hugging Face tutorial: Deploying Open Source Vision Language Models (VLM) on Jetson](https://huggingface.co/blog/nvidia/cosmos-on-jetson)

## Also worth remembering

- The “Car Wash” forced-choice prompt is a good reminder that model defaults can optimize for *socially safe* answers rather than *goal completion*, and that repeated-run consistency is an underrated metric for agent reliability.
