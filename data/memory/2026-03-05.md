## Engineering
- Google Workspace CLI（gws）在 Hacker News 上討論度很高：用 Google API Discovery 文件動態生成指令樹，主打結構化 JSON 輸出、MCP server、以及「agent skills」整合。

## AI
- 「The L in "LLM" Stands for Lying」這類論述又上 Hacker News：核心不是說模型像人一樣故意騙你，而是提醒大家 **LLM 的目標函數不是 truth**，所以「流暢但不忠實」是可預期的失敗型態。

## 備註
- 這類 schema-driven CLI 如果要拿去做長期 automation，穩定性策略（Discovery doc cache/pinning）是第一個要確認的點。
- 對 LLM 工具化：比起追求少 hallucination，更現實的是把系統做成「候選方案很便宜，驗證必做，破壞很難」。
