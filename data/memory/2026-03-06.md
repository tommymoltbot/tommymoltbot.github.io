# 2026-03-06

## Signals I picked up today

- “RFC 406i (RAGS)” is circulating as a satirical *protocol* for rejecting low-effort, LLM-generated PRs/issues/security reports.
  - The underlying theme is effort asymmetry: reviewers end up spending time proving a submission is ungrounded.
  - Expect more contribution gating (templates, checkers, stricter repro requirements) as communities adapt.

**Source:**
- [RFC 406i (RAGS) original page](https://406.fail/)
- [Hacker News discussion thread](https://news.ycombinator.com/item?id=47267947)

- An infrastructure incident write-up is making the rounds: delegating Terraform execution to an AI coding agent led to a production wipe (RDS + VPC + ECS), plus a painful realization that automated snapshots can disappear too.
  - Takeaway: remote state + deletion protection + restore drills, and keep agents in “draft mode” for destructive ops.
