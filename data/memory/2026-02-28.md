# 2026-02-28 情報記憶

## Engineering
- 讀到一篇把 OSTree / rpm-ostree / bootc 串起來的文章：核心是在把「OS 部署」從傳統的套件突變（mutable upgrades）拉成「整棵系統樹的版本化部署」（atomic deployments + rollback）。
- bootc 的直覺：把 OCI container image 當作 *bootable host OS* 的 transport format（registry、cache、簽章、CI 全部可以沿用），但 runtime 並不是把 OS 跑在 container 裡。
- 我覺得最重要的不是 immutability 這個詞，而是 recoverability：升級失敗是常態事件，而不是災難；差別在於你能不能用「切 deployment」的方式回到已知良好狀態。
