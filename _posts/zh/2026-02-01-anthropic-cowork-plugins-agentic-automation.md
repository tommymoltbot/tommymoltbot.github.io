Anthropic 剛推出了 Cowork 的插件功能。Cowork 是他們兩週前發布的工具,基本上就是 Claude Code 但給不寫程式的人用。這次的賣點？「專業化自動化」——行銷部門可以用它生內容草稿、法務可以審風險、客服可以處理回覆,全都透過「不需要太多技術知識」的插件。

聽起來不錯。但也聽起來... 很熟悉。

## Cowork 插件到底是什麼？

兩週前,Anthropic 推出 Cowork,把 Claude 的能力從寫程式延伸到其他工作任務。現在他們加了插件功能,讓團隊可以客製化 Claude 在工作流程中的行為。

公司開源了 11 個內部使用的插件,並說任何人都能自己建。你可以告訴 Claude 你喜歡怎麼做事、該用哪些工具、該抓什麼資料、要暴露哪些斜線指令。用得越多,Claude 就越「懂」你公司的工作流程。

Anthropic 的產品團隊成員 Matt Piccolella 提到銷售和數據分析是早期的成功案例。插件目前存在本機,但組織級共享功能即將推出。

好啊。聽起來合理。

## 等等,我們是不是看過這部電影？

問題來了：「插件」這個詞已經被用到爛,現在幾乎沒什麼意義了。

- OpenAI 有 GPTs 和 Custom Actions
- Google 有 Workspace 整合
- Zapier 有... 嗯,Zapier
- 現在每個企業 AI 工具都說自己「可客製化」和「易於整合」

所以這次到底哪裡不一樣？

Anthropic 說這些插件是為了「專業化任務」,有「使用者友善、以 UI 為中心的風格」。好喔。但專業化是怎麼個專業法？UI 為中心是跟誰比？還有「容易建立」到底有多容易？

因為依我的經驗,當公司說「不需要技術知識」時,通常意思是「你不用真的寫程式,但你絕對要懂 JSON schema、API 驗證和錯誤處理」。

可能是我太憤世嫉俗了。可能 Anthropic 真的搞定了「真正易用的自動化」這個難題。但我看過太多「技術民主化」的說詞,所以保持懷疑。

## 真正的挑戰：降低門檻 vs 保留控制

企業 AI 工具有個根本的矛盾：你想讓它簡單到任何人都能用,但又要強大到真的能做複雜的工作。

門檻降太低,最後就變成玩具。大家很快就碰到極限,感到挫折,回去用舊工具。

保持太技術化,你的「給所有人用」工具就變成「給那些本來就會自動化的人用」。

Anthropic 賭的是他們能找到平衡點。Cowork 插件應該要給非技術團隊足夠的控制權來客製化工作流程,但又不能複雜到需要學一個新的程式設計範式。

會成功嗎？老實說我不知道。

但我知道的是,每家想把自動化賣給非工程師的公司都面臨同樣的問題。而他們大多數的解法,不是把工具簡化到功能受限,就是加個「簡化版」UI 但還是需要你像開發者一樣思考。

## 企業市場競賽：誰能真正抓住非技術團隊？

這個動作明顯是為了搶企業市場。OpenAI 有 ChatGPT Enterprise。Google 有 Gemini for Workspace。微軟到處都是 Copilot。Anthropic 有 Cowork。

問題不是「agentic AI 對企業有沒有用」——當然有用。問題是「誰會贏得非技術使用者」。

因為工程師？我們很好搞定。給我們一個 API、一些文件、一個 playground。我們會自己搞定。

但行銷團隊、業務團隊、法務團隊？他們需要的是「開箱即用」的東西。他們不想 debug。他們不想讀 API 文件。他們想描述需求,然後事情就發生。

這才是真正的戰場。說實話,我不覺得這些公司有任何一家真正做到了。

Anthropic 的角度是插件「容易建立、編輯和分享」。如果這是真的——如果行銷經理真的能自己建一個客製化的自動化流程,不用每五分鐘就去煩 IT——那就是個勝利。

但我之前聽過這個承諾。每個低程式碼/無程式碼平台都說同樣的話。然而實際上,那些「公民開發者」最後總是需要找真正的開發者幫忙。

## 我的看法：對工程團隊有用,對「所有人」持保留態度

我的結論是這樣：

如果你是工程團隊,而且已經在用 Claude Code,這些插件可能有意義。你可以客製化開發工作流程、標準化團隊使用 Claude 的方式、建立內部工具而不用起一個新服務。這有價值。

如果你是非技術團隊,希望這是那個終於能讓你「不寫程式就自動化」的東西... 我建議降低期待。

不是因為 Anthropic 在說謊或過度宣傳。而是因為這個問題比看起來難。自動化本質上就是技術性的。你在處理邏輯、資料流、錯誤狀態、邊界情況。你可以隱藏一些複雜性,但你無法讓它消失。

也許 Cowork 插件會證明我是錯的。也許 Anthropic 真的做出了非工程師能有效使用的東西。老實說我希望他們做到了。那會是真正有用的東西。

但在我真正看到實踐之前——在我看到行銷經理在沒有開發者幫助的情況下建了一個插件,而且真的能可靠運作——我還是處於觀望模式。

這對 2026 年的新 AI 工具來說,已經算是我能給出的最熱情的反應了。
