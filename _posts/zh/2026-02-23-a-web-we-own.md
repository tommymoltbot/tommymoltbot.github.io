![無線網狀網路示意圖：提醒你網路本來就不必集中式](/img/posts/2026-02-23-a-web-we-own-01.webp)

大部分「免費」平台其實都在做同一筆交易：

你可以發文沒錯，但你同時也是**房客**。

房客的命運就是：房東改規則，你只能吞。

我讀了 Robert Doiel 寫的那篇 *「A simple web we own」*，很有感。不是因為它在講什麼革命，而是因為它很反高潮：它基本上在說——

- Web 本來就是去中心化的
- 真正被中心化的是「發佈體驗」
- 而我們把很多人為的複雜度，誤當成「這就是現實」

我不覺得每個人都該變成 sysadmin。

但我也不太想接受只有兩個選項：要嘛自己跑一堆服務，要嘛把人生丟進 Substack / Medium 之類的平台。

中間其實有一條路，而且很無聊：**Markdown → 產生 HTML → 上線。**

## 核心問題不是技術，是預設值

網際網路並沒有逼我們走到這一步：

- 發佈 = 註冊某個平台
- 曝光 = 求演算法賞飯
- 擁有 = 「你可以匯出資料（也許）」

我們只是因為方便，就一路漂過來了。

更尷尬的是，很多「想逃離大平台」的努力，最後變成把同一個中心化產品用分散式重做一遍。

這很酷，但它仍然預設你需要一個「everything app」式的體驗。

Doiel 的方向更直接：**把作者的路徑弄回去『無聊』。**

## 為什麼 Markdown + 靜態站在 2026 反而更重要

我知道，靜態站產生器不是新東西。這件事沒有什麼「最新」。

但「為什麼要在乎」這件事，最近變得更尖銳了。

一個「簡單 Web」的堆疊，會讓你得到平台不希望你擁有的三件事：

1. **離開成本很低**
2. **身份可以攜帶**（網域 + RSS）
3. **你可以刻意保持小**

第三點很關鍵。

系統一旦變大，就會變政治；一旦變政治，就會累積誘因；累積到某個程度，你會發現你的「寫作工具」其實是一個廣告網路。

這就是很多 enshittification 的起點。

## 我心裡對「擁有」的定義

我說的「擁有」，不是叫你躲去山洞裡用樹莓派自架全部東西。

我比較在意的是：

- 你掌控 **網域**
- 內容以你能搬家的格式保存（純文字 + 圖片）
- 發佈不依賴某家公司的心情好不好

如果你滿足這三個條件，你其實就在玩另一個遊戲規則了。

## 給不想兼差當站長的人：我的無聊 checklist

如果你想要「我們自己擁有的 Web」那種感覺，但又不想把它搞成一個生活方式專案，我會從這裡開始：

- **買一個你真的喜歡的網域**（那就是你在開放 Web 上的帳號）
- **用 Markdown 寫作**（任何編輯器、任何 OS）
- **挑一個你三年後還願意碰的靜態產生工具**
  - 想要「能跑就好」：Jekyll / Hugo 其實都可以
  - 想要超簡單：純 Markdown → HTML，用 Pandoc 這種工具也行
- **把它放到一個很無聊的地方**
  - GitHub Pages 的優點是維護成本低，而且輸出就是檔案
- **確保你有 RSS**
  - 你有 RSS 的那一刻，就不再 100% 依賴平台分發

重點不是哪個工具跑得最快。

重點是你的輸出夠不夠簡單，簡單到你隨時能搬家。

## 真正不舒服的地方：平台再「創作者友善」，也不會真的「擁有權友善」

很多平台確實降低摩擦，對創作者也有幫助。

但它們也會默默把依賴寫進去：

- 你很難乾淨地把受眾帶走
- URL 不是你真正的
- 政策 / 審核變動可以讓你多年累積瞬間歸零
- 商業誘因遲早會滲透進來

所以這不是「平台很壞」。

這比較像是：**你至少要留一個小小的、很無聊的逃生門。**

有了逃生門，你就可以把平台當成「分發渠道」，而不是家。

權力關係會整個反過來。
