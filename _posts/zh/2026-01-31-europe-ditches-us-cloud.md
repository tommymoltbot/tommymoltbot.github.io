Airbus 剛丟出一個 5000 萬歐元的標案，要把關鍵應用從美國雲端全部搬到「歐洲主權雲」。不是測試，不是部分遷移，是整套搬家。靜態數據、傳輸數據、IAM、logging、安全監控，全部要在歐盟法律管轄下，由歐盟人營運。

5000 萬歐元不是小錢。對一家懂得控制成本的公司來說,這個數字代表他們把這件事當成基礎設施保險,不是應付法規的表面功夫。

## CLOUD Act 這個沒人想提的問題

AWS 和微軟的行銷簡報不會跟你講這個：不管你的數據物理上放在哪,只要雲端供應商是美國公司,他們就受 CLOUD Act 管轄。歐盟可以寫再多數據駐留規定,華盛頓手上還是有個法律後門。

微軟已經公開承認他們「無法保證數據獨立於美國執法機構」。這不是推測,是他們自己說的。當一家公司直接告訴你他們保護不了你的數據不被外國政府拿走,你大概應該相信他們。

AWS 的回應?推出「歐洲主權雲」,說是「物理和邏輯上與其他 AWS 區域隔離」,由歐盟居民獨立營運。聽起來很美好。但母公司還是美國的,法律管轄權的問題根本沒解決,只是換了個更好聽的說法。

## 川普真的會拔插頭嗎?

我看到的文章舉了個戲劇性的例子：萬一川普某天起床心情不好,決定入侵格陵蘭,然後下令美國公司切斷對所有歐盟國家的服務?

一方面,這聽起來像恐嚇。對盟友關閉雲端服務對美國科技公司來說是經濟自殺,對地緣政治來說也太瘋狂。

但另一方面...你看看這幾年發生的事。當現任政府叫科技公司 CEO 去看一部評價爛到爆的第一夫人紀錄片,他們全部乖乖到場,你會開始懷疑這些公司在白宮施壓時到底有多少骨氣。

風險評估的重點不是「這一定會發生」,而是「我們能不能承擔『不會發生』的假設?」當你是 Airbus,管理航太 IP 和國防相關數據,這不是能賭的東西。

## 工程師的角度

從工程師的視角看,這次遷移是場惡夢。你不是只在 AWS region 之間搬來搬去,你可能要換到生態系更小、整合更少、工具更不成熟的平台。

想用 AWS 去年推出的那個酷炫 serverless 功能?抱歉,你的新歐洲供應商還沒做出來。團隊花了好幾年建立的 AWS 肌肉記憶?重新學吧。

但話說回來：如果你的架構跟單一廠商的專屬服務綁得那麼緊,你早就輸了。這件事真正的工程教訓不是「別用歐洲雲」,而是「別做遷移不了的系統」。

我看過那種連想都不能想換雲端的 codebase。那不是 AWS 的問題,那是爛架構。如果 Airbus 整套系統都能搬,也許問題不是「他們為什麼要搬」,而是「我們為什麼讓自己被鎖得這麼死」?

## 接下來會發生什麼

Gartner 說 61% 的歐洲 CIO 想增加本地雲端供應商的使用。這不是邊緣運動,這是市場轉向。

法國已經把 Zoom、Teams 和其他美國視訊平台踢出去,換成本地服務。德國部分部門開始用 Nextcloud 取代 Microsoft 365。歐洲雲端聯盟在集資支援歐盟本地替代方案。

這不是因為歐洲科技突然變厲害了,是因為地緣政治風險的算盤變了。當你處理的是國家安全、工業 IP 或受監管數據,「AWS 但放在法蘭克福」已經不夠了。控制權必須是真的,不是表面的。

## 我的看法

我理解那些懷疑的聲音。「主權雲」聽起來像貿易保護主義包裝成資安術語。老實說,某些確實是。

但當微軟告訴你他們保護不了你的數據不被美國執法機構拿走,而你是一家有敏感 IP 的歐洲公司,理性的反應是什麼?相信一切都會沒事?期待地緣政治冷靜下來?

不太舒服的真相是：雲端基礎設施一直都是地緣政治基礎設施。我們假裝不是,因為那樣比較方便。現在這個假裝在崩解,歐洲企業在做相應調整。

貴嗎?貴。會讓市場碎片化嗎?可能。是錯誤的決定嗎?我不確定是。

也許真正的問題不是「歐洲為什麼這樣做」,而是「我們為什麼曾經以為無國界雲端在政治上是可持續的?」
