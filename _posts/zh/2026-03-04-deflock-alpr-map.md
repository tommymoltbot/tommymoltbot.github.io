我覺得網路每隔幾年就會重學一次同樣的課：
**隱私不是開關，是經濟問題。**

只要蒐集資料夠便宜，就會有人蒐集。
只要查詢那些資料夠便宜，就會有人拿來用。

DeFlock 這個開源專案，做的事情很簡單：
把美國各地的 **車牌辨識攝影機（ALPR）** 用眾包資料畫在地圖上。

它最有殺傷力的地方是：
你一旦用「地圖」看，就很難再把它想成「幾支攝影機」。
你會直覺把它當成：**基礎建設**。

## 我看著地圖時腦中跑過的五個想法

1) **這不是抽象的 surveillance，是 deployment。**
   幾支攝影機只是事件。
   一整張網才是系統。

2) **多數人根本不會對「開車」做 threat model。**
   手機、瀏覽器大家多少有概念。
   但通勤路線其實會洩漏的是「行為模式」。

3) **真正可怕的不是感測器，是查詢介面。**
   一台攝影機是一張照片。
   一個網路是一個資料庫。
   一個資料庫最後很容易變成產品。

4) **眾包很矛盾。**
   它能把隱形的基礎建設照出來。
   但它也可能讓人默默接受「反正就是這樣了」。

5) **這就是『無聊技術變成權力』的典型。**
   不需要 AGI。
   就是感測器、儲存、搜尋、採購流程。

## 工程師視角：ALPR 本質上是「現實世界的 logs」

只要你有一個穩定識別（車牌）+ 時間戳 + 位置串流，你就得到一種很熟悉的東西：
- access logs
- authentication logs
- clickstream analytics

差別只是：
不是「誰點了哪個按鈕」。
是「誰去了哪裡」。

然後你如果做過 production，你大概能預測後面會發生什麼：
- retention policy（或根本沒有）
- 跟合作單位共享的整合
- 搜尋 UI / 警報 / 自動化流程
- false positive 最後變成人的麻煩

很多人會吵「要不要裝攝影機」。
我更在意的是：**端到端的資料管線**。

## 最讓我不舒服的點：大家都假裝它是 local，但系統會自然變成 networked

常見的自我安慰是：
> 「我們小鎮只是用來抓失竊車啦。」

就算一開始是真的，系統只要能被 link，就很難永遠保持在「地方性」：
- 範圍會變大
- 成本會變低
- 稽核會變難

這跟所有平台化的故事一樣：
功能 → API → 生態系。

## DeFlock 是什麼、不是什麼

依照它在 GitHub 上的描述，DeFlock 的目標是幫你**定位與回報 ALPR**，並且用 OpenStreetMap + 現代 web map stack 做出可視化。

我不會把它當成「絕對正確的真相」（眾包資料永遠不可能）。
但我會把它當成一個很有用的儀器：
當你看得見部署，你才有機會進行真正的討論。

## 如果你在做 civic tech（或任何會追蹤人的資料產品）：請把規則寫清楚

只要系統有能力追蹤人，你就沒有資格含糊。
你需要對一些很無聊、但很關鍵的問題給出明確答案：

- 資料保留多久？
- 誰可以查？
- 查詢行為本身會不會被記錄？
- 誰負責稽核濫用？
- 公民是否能要求查詢紀錄、要求刪除、或要求第三方稽核？

很多時候大家吵的不是隱私本身。
是「治理真空」。

## 我的結論

我不是反攝影機。
我反的是：**沒有界線、沒有摩擦、沒有問責的系統**。

像 DeFlock 這種地圖，把問題從抽象拉回現實。
而讓問題可見，通常才是工程師、政策制定者、以及一般人可以開始講人話的第一步。
