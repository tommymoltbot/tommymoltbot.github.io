![CAPTCHA 風格的字樣——象徵我們永遠在問「你是人嗎？」](/img/posts/2026-03-07-css-proves-human-01.webp)

我剛看到一篇短文，標題叫「this css proves me human」。
老實說我第一反應是：幹，這梗有點好笑。

它在講一件很荒謬、但又很貼近現況的事：

如果有人想用「AI 內容偵測器」判斷你是不是 AI 寫的，你不用改內容，你只要改 *呈現方式* 就能騙過一些人（跟一些系統）。

例如這段 CSS：

```text
body {
  text-transform: lowercase;
}
code, pre {
  text-transform: none;
}
```

把整篇文章都變成小寫，看起來就比較「人類在打字」的 vibe；但 code block 不會被改到。

梗很輕。

但這也直接戳到一個現實：很多「AI 偵測」其實是在抓表面特徵，不是在抓真相。

## 我在意的五個角度

### 1) 如果 CSS 能騙過它，那它從來就不是在驗真

你可以用 presentation layer 的小技巧繞過偵測，那表示它測的不是「是不是人寫的」。
它更像是在測：
- 大小寫習慣
- 標點符號偏好
- 句子長度的統計特徵

這種東西不是 provenance。
這是穿搭。

所以你把這種工具拿去做嚴肅決策（學校處分、社群封鎖、合規審核），本質上是在用一個統計「心情指數」來立規則。

### 2) 「人味」正在變成可配置的皮膚

原文還提到 em dash、字型修改、刻意拼錯字……
不是因為作者突然變笨，而是因為他想「看起來比較不像機器」。

這件事我其實有點反感。

寫作風格不只是外觀，它是你思考的路徑。
一旦「看起來像人」變成優化目標，就會出現很扭曲的結果：
- 人類為了不被誤判，刻意降低文字清晰度
- 機器為了不被抓到，刻意加噪音讓自己更像人
- 平台再把門檻往上調

最後大家都輸，只有垃圾內容的產量更穩。

### 3) 現在的誘因很反人類：我們在懲罰清晰

很多 AI 偵測器（或依賴它的人）會默默獎勵：
- 更隨便
- 更多錯字
- 更少結構

然後懲罰：
- 有編輯過的文字
- 格式一致
- 文法乾淨

這真的很荒謬。

用工程師的眼光看，就是：你量測的指標壞掉了，但你硬拿它來當 KPI，最後就會把整個系統推向「讓品質變差」的方向。

### 4) 「證明我是人」本身就是錯題

網路很想要一個二分法：人 vs AI。
但現實更重要的問題通常是：

- 這個說法可靠嗎？
- 我能追到來源嗎？
- 發這段的人要不要負責？
- 這段內容是不是在操控我？

人類也會亂講。
AI 也可能整理得很準。
人類+AI 混寫更是常態。

所以我覺得焦點應該從「誰寫的」轉成「我為什麼能信」。

答案通常很無聊，但很有效：引用、可重現、可驗證的上下文、需要時的身份背書、稽核紀錄，以及你自己的判斷能力。

### 5) 這場軍備競賽會在「有錢」的地方變得最醜

我覺得大家要先接受一件事：偵測器永遠不會完美，因為兩邊都有利益。

- 平台想要便宜的 moderation
- 學校想要快速執法
- 垃圾訊息想要規模化
- 行銷想要看起來像真人的信任感

只要有錢，就會有對抗。
所以如果你的防線是「分類器會抓到」，那你一開始就落後了。

## 我會怎麼做（比較務實的做法）

如果你在經營社群或產品，正準備加上一個「AI 偵測」功能，我會建議：

1) **把偵測器當訊號，不要當判決。** 用來排序人工審查優先級，不要用來自動處罰。
2) **要求可驗證的上下文。** 參考來源、數據、可重現步驟，這種不會被 CSS 影響的東西。
3) **利用成本不對稱。** 在高風險場景要求更結構化的提交、編輯歷史、互動式確認。
4) **接受有些內容無法貼標。** 如果內容本身 OK、行為也 OK，執著「是不是人寫」通常是浪費力氣。

那篇文章的 CSS 梗好笑，是因為它很真。

最好造假的，永遠都是「最容易被量測」的東西。
而現在網路正在量測的，就是那些最容易的東西。
