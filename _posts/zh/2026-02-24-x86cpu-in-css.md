網路上每隔一段時間就會冒出一種專案，專門用來挑戰你對「什麼算程式語言」的直覺。

這次的主角是：**用 CSS 寫出一個 x86 CPU emulator**。

對，就是那個 Cascading Style Sheets。作者說核心是 CSS-only（頁面上有 script tag 主要是提供 clock 讓它更穩更快；就算禁用 JS 也能跑，只是比較慢）。

很明顯這不是 practical 的東西，但我反而覺得它不只是搞笑，因為它逼你正面看一件事：**瀏覽器其實是一個很奇怪的計算平台**。

## 有意思的點（不是「哈哈 CSS」而已）

### 1) Browser 本來就藏著一個很重的 evaluation engine

做過大型 CSS 的人一定懂那種感覺：

- 你以為你在寫樣式
- 但瀏覽器在背後做的事情，很多時候已經像是在跑程式
- 然後某天你改一行，CPU 就被打滿，你開始懷疑人生

這個專案只是把那個「現實」拉到極致，變成一個作品。

### 2) 這是「平台能力邊界」的地圖

平常大家講「某某語言是圖靈完備」很多時候只是冷知識。

但在這個專案裡比較具體：

- container query、animation、selector 的規則變成「clock」
- style 本身變成 state storage
- rendering 變成輸出

這不是我們想要的軟體工程方式，但它告訴你：平台一旦允許足夠複雜的 state + transition，真的有人會拿來做怪事。

### 3) 限制越硬，創意越歪

我不希望任何人把 production 邏輯寫在 CSS。

但我很喜歡看這種東西，因為它證明：

- 只要平台有某種 side-channel 可以做 state 與狀態轉移
- 就有人會在上面蓋一台電腦出來

而且更討厭的是：它常常比官方文件更能讓你理解平台到底怎麼運作。

## 如果你是工程師，你可以帶走什麼？

幾個 boring 但有用的 takeaway：

- 「它是 declarative」不等於「它不會算東西」。
- 很多 performance cliff 就藏在「宣告式」背後那台龐大的 evaluation engine。
- security boundary 很重要：只要你允許一個複雜的 evaluation model，人就會用奇怪的方法繞過你的假設。

但老實說，這篇我寫到最後最想講的是：它很好玩。電腦本來就可以很好玩。
