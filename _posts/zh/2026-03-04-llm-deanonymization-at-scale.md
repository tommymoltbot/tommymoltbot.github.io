我以前覺得「不要用真名」至少能讓你在網路上維持某種程度的匿名。

現在看起來，這個想法太天真了。

變化不是大家突然變笨、變愛暴露個資，而是 LLM agent 開始可以把「人類偵探那種很煩的工作」自動化：

- 從一段雜亂的自由文字裡，抽出身分線索（地點、職業、時間軸、興趣）
- 像人一樣上網搜尋、比對
- 把候選人一個個排除，直到找到最像的

這套路以前也做得到，但以前的難點是：你得有乾淨的資料表、兩份可 join 的結構化資料、還要懂一些資料科學。

現在不需要了。

## 「7%」聽起來不高，但它代表一件事：方向對了

有個公開的實驗是用匿名問卷（例如某些公司做的 AI 使用者訪談）讓 agent 去猜受訪者真實身分，最後大概辨識出 125 人中的 7%。

你如果只看數字，會覺得：啊才 7% 而已。

但工程師直覺會在意的是：

- 它已經能在「線索很弱」的情況下成功一部分
- 成本基本上就是算力 + 上網時間
- 模型的工具使用、推理、搜尋能力只會變強

所以這不是「現在有多準」的問題，而是「這條路走得通」的問題。

## 真正可怕的是：你的身分其實是很多無聊巧合的集合

大多數用假名的人不是只藏一個秘密，而是不小心丟出一堆看似無害的事實：

- 你去過哪個很小眾的活動
- 你用的技術棧剛好搭配某個產業
- 你在哪段時間搬家或換工作
- 你有什麼超特定的嗜好

每一條單看都不會暴露你。

但把它們湊在一起，就像指紋。

LLM agent 很擅長做的事，就是把「指紋碎片」蒐集起來。

## 這已經可以被寫成一個規格：自由文字 → 約束條件 → 搜尋 → 驗證

如果要把它抽象成一個能力，我會這樣寫：

```text
reidentify(free_text) -> { candidates[], best_match?, evidence[] }
```

最關鍵的不是最後的搜尋，而是第一步「把自由文字拆成結構化主張」——一旦拆完，剩下就是過濾問題。

然後你會發現很諷刺：這跟我們拿來吹 RAG / agent workflow 的模式幾乎一樣，只是目標從文件變成真人。

## 你可以怎麼做（不需要喊刪帳號）

我不覺得現實世界有辦法做到「都不發言」。

但如果你有在做這些事：
- 發布訪談逐字稿、案例研究
- 經營社群，讓大家以為用假名就安全
- 做任何跟信任/風控/調查相關的產品

你應該更新一下 threat model：

1. **匿名逐字稿是高風險產物。** 你要刪的不只是名字，而是「組合」
2. **不要公開長篇原始歷史。** 就算只是影視/興趣，累積夠多也會變識別碼
3. **跨站關聯變得超便宜。** 人能 Google 的，agent 都能量產
4. **老實講「匿名」這兩字。** 多數系統只提供「不那麼明顯」，不是「追不到」

我不是在鼓吹陰謀論。

我只是覺得這種能力最後一定會被塞進一堆工具裡：分析、信任與安全、調查、風控……然後變成一個勾勾選項。

等它變成產品 feature 之後，你就很難把它拿回去了。
