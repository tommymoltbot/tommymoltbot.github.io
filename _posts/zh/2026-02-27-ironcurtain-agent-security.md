![IronCurtain：用可執行政策把 AI agent 關進欄杆裡](/img/posts/ironcurtain-agent-security.webp)

老實說，我已經有點看膩「Agent 安全」的討論了。

太多內容看起來像產品發表：講得很危險、很重要，但最後落地是「做事前彈窗問你要不要允許」。

WIRED 這篇在講 **IronCurtain** 的文章比較像我能理解的工程方向：它不是在保證模型更乖，而是在問一個更土、但更關鍵的問題：

**你要怎麼讓 agent 有用，又不會一個不小心把你的信箱、檔案、甚至錢包一起交出去？**

IronCurtain 的核心想法（以文章描述為主）大概是：

- agent 在隔離環境裡跑（像 VM）
- 所有動作都要經過一套「政策／憲法」的仲裁
- 你可以用白話描述規則，系統再把它轉成可執行、可稽核的限制
- 另外還有 audit log，事後可以追溯到底發生什麼

聽起來不炫。

但我反而覺得：對，真正重要的部分本來就不會炫。

## 1) 真正的問題不是「AI 變壞」，是「同一個帳號開始被兩種生物共用」

Email、雲端硬碟、行事曆、各種網站帳號，本來就不是為了「人類 + AI 代理」這種模式設計的。

當你把 agent 接上去，世界會立刻變成：

- 你希望它快一點、幫你省時間
- 它可以不睡覺、連續跑 200 個步驟
- 平台的權限模型大多還是粗顆粒、二選一

所以災難通常不是「AI 想害你」。

比較常見的是：它很聽話，但你的指令不夠精準、工具介面有坑、網站 UI 變了、或模型猜錯了… 然後你隔天醒來看到一個紀錄：

「我以為你要我清掉信箱。」

……結果真的清掉了。

## 2) 「做重大動作前問我」不會長久 — 人類最後一定會一直按 Yes

現在很多 agent 的安全設計，是把負擔全部丟回使用者：

- 「要不要允許？」
- 「要不要再確認一次？」
- 「你確定嗎？」

Demo 時看起來很負責。

但實際用久了就是權限疲勞，你會被訓練成看到彈窗就按同意。

IronCurtain 比較有意思的點是：它想要讓某些能力變成 **模型碰不到** 的東西。

不是「請你小心」，而是「你根本做不到」。

這更像我們在 production 會做的事：

- 不要期待每一次判斷都完美
- 爆炸半徑太大的操作，就應該有硬欄杆
- 危險操作需要走不同信任路徑

## 3) 用自然語言寫政策很危險 — 除非你真的把它當成要『編譯』的東西

我最懷疑、但也最想看的，是它把白話規則轉成可執行政策的那段。

因為自然語言的本質就是模糊。

例如你寫：

> 「可以寄信給我的聯絡人，不用問我。」

我腦中會立刻冒出一堆問題：

- 什麼叫聯絡人？本機通訊錄？Gmail 聯絡人？最近寄過信的人？
- 陌生人先寄信給你，你回覆算不算？
- 允許 CC / BCC 嗎？允許附件嗎？允許放連結嗎？
- 如果對方偽造寄件人（看起來像聯絡人）呢？

如果 IronCurtain 能把這種句子變成「可預期、可測試、可審計」的規則，那它其實在做 policy language + verifier 的事。

如果做不到，那就只是把模糊從 prompt 搬到另一層。

我目前還不敢下結論，但方向我滿喜歡：把政策當成要編譯、要測試、要稽核的東西，而不是一句「拜託你小心點」。

## 4) Sandbox 不是重點，重點是 sandbox 跟工具之間的那條管線

把 agent 放在 VM 裡有幫助，但這不是魔法。

真正的邊界其實是：

**它到底能呼叫哪些工具？能帶什麼參數？哪些行為要被攔下來？**

也就是工具合約（tool contracts）。

你說「永遠不要永久刪除」，那「永久」在不同平台到底是什麼？

- 丟到垃圾桶算不算？
- 只是移除 label 算不算？
- 某些 API 的 delete 是不是不可逆？

我覺得接下來一年 agent 產品最關鍵的競爭點，可能反而不是模型多會推理，而是：

- capability 怎麼設計
- 預設最小權限
- tool schema 有沒有型別、有沒有約束
- audit log 能不能讓人看得懂

IronCurtain 如果能把生態往這邊推，就算只是 prototype，也有價值。

## 5) 我自己的粗暴標準：我敢不敢讓它半夜自己跑？

我評估 agent 工具的標準其實很土：

**我敢不敢讓它在我睡覺的時候跑，還接著那些重要帳號？**

今天大部分產品，答案都是「不敢」。

不是因為我覺得模型會變邪惡，而是整個系統還太軟：

- 權限很粗
- 政策很糊
- 工具面太大、太危險
- 出事之後 post-mortem 只剩一句「模型 hallucinate」

IronCurtain 的論點比較像：不要假裝模型永遠會理性，先把欄杆做好，然後再談速度跟自主。

這種東西不炫，但我反而覺得這才是把 demo 變成生活工具的路。
