最近看到一篇文章在 Hacker News 上爆了,188 個讚、165 則留言。標題很簡單：「兩種 AI 使用者正在出現,差距驚人。」我點進去看完,第一個反應是——靠,這不就是我這幾個月看到的狀況嗎?

作者 Martin Alderson 把現象講得很直白：現在用 AI 的人分成兩群。一群是 power users,開 Claude Code、跑 terminal、用 MCP、搞 skills。另一群呢?還在用網頁版 ChatGPT 聊天。

聽起來好像沒什麼,但實際差距有多大?文章舉了個例子:有個非技術背景的財務主管,用 Claude Code 把一個 30 張 Excel 表的複雜財務模型,幾個 prompt 就轉成 Python 了。轉完之後呢?他突然有了整個 data science 團隊的能力——跑 Monte Carlo 模擬、串外部資料源、做 web dashboard。

這就是兩種人的差距。一邊是用 Excel 慢慢調公式的人,另一邊是把整個工作流程用程式重構的人。

## 微軟賣 Copilot,自己用 Claude

文章裡最諷刺的一段是這個:微軟內部團隊正在導入 Claude Code,即使他們有 M365 Copilot、即使他們是 OpenAI 大股東、即使 Copilot 對內部員工幾乎零成本。

為什麼?因為 Copilot **真的很爛**。

作者說得很直接:「Copilot 感覺像是 ChatGPT 介面的劣質仿製品。」它的 agent 功能跟 CLI coding agent 比起來?「完全是個笑話。」

更慘的是,很多企業裡 Copilot 是**唯一被允許的 AI 工具**。IT 政策鎖死,你想用別的?要嘛被開除,要嘛花一堆力氣去申請採購。結果就是公司高層用著這爛工具,然後下結論:「AI 根本沒用。」

然後他們就把預算砸給顧問公司,花大錢搞一堆沒什麼用的東西。

## 大企業的死法

文章把企業的問題歸納成三個致命組合:

1. **鎖死的 IT 環境**:連跑個 script 都不行,VBA 如果你運氣好還能用一下
2. **沒有 API**:核心系統都是老古董,根本沒有「內部 API」可以串
3. **工程團隊外包或孤島化**:沒人能幫你建一個安全的 agent 沙盒

這三個加起來是什麼?就是大企業**無法使用真正有用的 AI 工具**。

我覺得這個觀察很準。不是說大企業的安全考量不對,你當然不能讓人隨便拿個 coding agent 去操作生產資料庫。但問題是,當你的 IT 政策把所有可能性都鎖死,工程團隊又沒有能力幫你建一個安全的環境時,你就卡住了。

小公司呢?沒這些包袱。他們用新的 SaaS 工具,這些工具大多有現成的 API。員工可以自己裝 Python、跑 local script、用 Claude Code。結果就是 10 個人的團隊,生產力可能比 1000 人的企業部門還高。

這是我見過的第一次,小團隊真的可以在純執行力上**屌打**大公司。

## 財務人員不需要工程師了

文章裡有個細節我特別喜歡。作者說他看到很多**非技術背景的人**在用 Claude Code,而且用得很好。尤其是財務角色。

為什麼?因為 Excel 對財務分析來說,真的太弱了。

當你習慣了 Python 的彈性之後,Excel 那套「拉公式、複製格式、手動調整」的流程就顯得超級侷限。但以前沒辦法,財務人員不會寫 code,請工程師又太慢、溝通成本又高。

現在呢?Claude Code 幫你把 Excel 轉成 Python,然後你直接問它:「跑個 Monte Carlo 模擬」、「把這個資料視覺化」、「串進這個 API」。它就幫你做了。

這才是真正的生產力革命。不是那種 PPT 上寫的「提升 30% 效率」,而是**根本改變工作方式**。

## 未來會怎樣

作者最後講了幾個觀察,我覺得滿有道理:

**第一,真正的進步是自下而上的。**  
不是什麼「AI 轉型策略」或「數位轉型計畫」,而是小團隊自己決定試試看,然後做出成果。他們最懂流程,不需要外包工程師來猜。

**第二,有 API 的公司會贏。**  
不管是 read-only 的 data warehouse,還是完整的 API 化核心流程,能讓 agent 串接的公司會跑得比別人快。

**第三,老牌 SaaS 公司危險了。**  
如果你的產品是十幾年前做的,API 是後來硬加的、只給開發者用的,那你可能會變成瓶頸。員工會想繞過你,因為你跟不上他們的需求。

**第四,小團隊的時代來了。**  
歷史上從來沒有哪個時代,一個 10 人團隊可以這麼輕鬆地打敗 1000 倍規模的公司。

## 我自己的看法

這篇文章講的東西,我在身邊真的有看到。

有些小公司的工程師跟我聊,他們現在用 AI 輔助開發的速度,比以前快太多了。而且不是那種「複製貼上 ChatGPT 的 code」,而是真的把 Claude Code 當成一個**隨時能討論的資深同事**在用。

反觀一些大企業的朋友,他們還在用那個爛 Copilot,或是根本連 AI 工具都碰不到,因為 IT 政策不允許。

差距會越拉越大。而且我覺得這不只是工具的問題,是**組織結構的問題**。大企業的 IT 政策、採購流程、安全審查,這些東西在沒有 AI 的時代或許還能忍受,但現在?它們變成了**致命的劣勢**。

小公司沒有這些包袱,可以快速試錯、快速調整。大企業呢?等你的審批流程走完,人家早就迭代三輪了。

所以我覺得,接下來幾年會很精彩。要嘛大企業真的開始改組織、改流程、給員工更多自由度,要嘛就等著被小團隊吃掉市場。

反正對工程師來說,現在是個很有趣的時代。你可以選擇待在大公司慢慢等死,也可以去小團隊感受那種「10 個人當 1000 人用」的快感。

我知道我會選哪個。
