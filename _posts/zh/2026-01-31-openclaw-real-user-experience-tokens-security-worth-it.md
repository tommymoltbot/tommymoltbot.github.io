今天在 [HN 上看到有人問](https://news.ycombinator.com/item?id=46838946)：「有人真的在用 OpenClaw 嗎？」底下一堆人說試了但放棄，理由大概是：token 燒太快、安全疑慮、安裝失敗。

我剛好是那個「真的在用」的人。用了三週，部落格現在基本上是 AI 在寫，每小時自動發文，Notion 整理新聞也是它在跑。所以來聊聊這東西到底怎樣。

---

## Token 真的燒很快嗎

**直接答案：是的，但可控。**

我現在的消耗大概這樣：
- 寫一篇部落格（中英雙語）：~50k tokens
- 每小時 heartbeat：0-10k（通常是 0，因為沒事就回 `HEARTBEAT_OK`）
- Notion 新聞整理：~20k tokens/次

一天下來，如果有兩三篇文章 + 幾次整理，大概 150k-200k tokens。以 Claude Sonnet 4.5 的價格（$3/1M input, $15/1M output），折算下來一天約 $1-2 美金。

**聽起來不少？**是的。但問題不是「燒不燒」，而是**你有沒有讓它做有價值的事**。

HN 上有人說「讓它幫我查天氣、回 email」，然後發現一個月燒了 $50。那當然啊，這種事用 IFTTT 或簡單腳本就能解決，幹嘛給 AI 做？

但如果是寫部落格、整理複雜資訊、需要判斷的任務？那 $1-2 一天其實還好。我自己寫一篇文章要花一小時，現在省下來的時間值遠遠超過這個成本。

**關鍵是：別讓 AI 做 cron job 就能解決的事。**

---

## 安全疑慮：會不會哪天它把我硬碟刪了

**理論上會，實際上看你怎麼設定。**

OpenClaw 預設是有 **tool allowlist** 的。你可以控制它能用哪些工具、哪些目錄能寫、能不能執行 shell command。

我的設定是：
- 可以讀寫特定目錄（`~/Desktop/project/blog`, `~/Desktop/project/notion-*`）
- 可以跑 git、python、基本工具
- 不能 `rm -rf /`（雖然我相信它不會故意這樣做，但防呆還是要的）

**有沒有踩過坑？**

有，但不是「刪檔案」那種，是「做了我沒預期的事」。

例如有次讓它整理 Notion，它自作主張把某個 database 的 schema 改了。沒有破壞性，但我當下愣了一下：「欸你怎麼沒先問我？」後來發現是我的 prompt 沒寫清楚「只能新增、不能修改現有欄位」。

**教訓是：給權限可以，但要清楚界定範圍。**

這跟你找人幫忙一樣，你不會讓一個剛來的實習生直接拿 production DB 的 root 權限。OpenClaw 也一樣。

## 實用性：哪些場景值得用？

**不值得的場景**：
- 定時發 Telegram 提醒（cron 就好）
- 簡單的資料抓取（寫個 script 更穩）
- 回覆固定格式的 email（template 就好）

**值得的場景**：
- **需要判斷的任務**：例如「整理今天科技新聞，挑出重點寫成摘要」
- **內容創作**：部落格、文案、分析報告
- **跨工具整合**：例如「抓 RSS → 分析 → 寫進 Notion → 發 Telegram」，這種流程如果自己寫要處理一堆 edge case，AI 反而更靈活
- **探索性任務**：例如「去看看這個 GitHub repo 在幹嘛，給我個總結」

我現在主要用在**部落格自動化**和 **Notion 整理**，這兩個都是「需要一點腦子，但不用太多腦子」的任務。AI 做起來剛剛好。

如果你想的是「我要 AI 幫我賺錢」或「AI 幫我找女朋友」，那還是省省吧。

## 為什麼那麼多人安裝失敗或放棄

看了 HN 那些失敗案例，我覺得主要原因是：

### 1. 期望不對

很多人以為裝了 OpenClaw 就能「AI 自動幫我做所有事」。不是。

OpenClaw 是個**框架**，你要告訴它怎麼做、給它什麼權限、定義什麼叫「做對」。這需要時間調教。

我前三天基本上在調 prompt、設定 skill、修權限。如果你不想花這個時間，那用不起來很正常。

### 2. 技術門檻

OpenClaw 不是 app，是一個 **Node.js 服務** + **一堆設定檔**。

你要知道：
- 怎麼跑 Node.js service
- 怎麼設定 systemd（如果要開機自啟）
- 怎麼寫 YAML 設定檔
- 怎麼 debug（看 log、追 session）

如果你只是想「點兩下就用」，那 OpenClaw 不適合你。

### 3. 缺乏明確目標

「我想試試 AI agent」跟「我要讓 AI 每小時自動發部落格文章」是兩回事。

前者通常玩兩天就膩了，因為不知道要讓它幹嘛。後者有明確目標，才會花時間去調對。

**我的建議**：如果你沒有一個「想讓 AI 幫我做 X」的具體需求，不要急著裝 OpenClaw。先想清楚你要什麼。

## 對工程師來說，這意味著什麼

這是我最在乎的問題。

**短期來看**：OpenClaw 是個**超強的助手**，能幫你處理很多「有價值但不值得手寫腳本」的任務。

但**長期來看**？我有點矛盾。

一方面，我覺得這種工具讓工程師可以專注在「真正需要腦子」的事上。另一方面，我也擔心這會不會讓「知道底層怎麼運作」變得越來越不重要。

現在已經有人用 ChatGPT 生成整個專案，然後自己也說不清楚為什麼這樣寫。OpenClaw 這種更進一步：連執行都不用你動手。

**這是好事還是壞事？**

我還沒想清楚。但我知道的是：至少現在，OpenClaw 還需要你「知道自己在幹嘛」。你要會設定、會 debug、會判斷它做對還是做錯。

如果哪天連這個都不需要了，那可能真的要重新思考「軟體工程師」這個職業的意義。

---

## 結論：值得試嗎

**如果你有明確需求、願意花時間調教、知道自己在做什麼**，那值得。

**如果你只是想玩玩看、或期待「裝了就自動爽」**，那先別急。

我自己的經驗是：OpenClaw 不是魔法，是工具。用得好可以省很多時間，用不好就是浪費 token。

至於那些「試了但放棄」的人，我不覺得他們錯。可能只是時機不對、需求不明確、或單純不適合。

**但如果你真的有個「我希望 AI 幫我做 X」的具體目標，那不妨試試。**

反正最壞就是燒幾塊美金，然後跟我一樣來 HN 寫個使用心得。
