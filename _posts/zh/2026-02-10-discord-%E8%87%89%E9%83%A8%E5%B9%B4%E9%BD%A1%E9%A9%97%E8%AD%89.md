Discord 說下個月起要做 **全球年齡驗證**：帳號預設會被套用「適合青少年的體驗」，除非你能證明你是成年人。

我第一個反應不是道德恐慌。
比較像是工程師的那種「嗯……果然走到這一步了」。

因為這種功能的本質其實很直白：平台想要 **合規（兒少安全）**，但又不想讓留存直接爆炸。

所以你會看到一個分層的機制：

- **年齡推論模型**：用帳號 tenure、裝置、活動模式去猜你是不是大人
- 如果猜不出來 → 叫你做 **裝置端的臉部年齡估計**
- 如果你想要完整權限又過不了推論 → 走 **第三方證件驗證**

聽起來很合理。
真正麻煩的地方是：它把「證明你是成年人」變成一種新常態。

## 1) 這不是政策調整，是一種新的「信任稅」

官方說法一定會是：大多數成年人不需要做任何事。

但使用者感覺到的流程是：

如果系統沒辦法推斷你是大人，你要嘛：

- 接受被限制的體驗（進不了 age-restricted server、一些功能被卡）
- 要嘛付出信任稅：自拍影片或上傳證件

就算他們說「臉部估計在裝置端跑、影片不離開手機」「證件會很快刪除」，
你仍然被要求進入一個 **曾經在這個產業出過事的驗證管線**。

使用者不會拿白皮書來審核。
他們用直覺判斷：

> 「我今天做了，明天還能反悔嗎？」

而這就是信任負債。

## 2) 最可怕的不是青少年繞過，而是成年人被誤殺

青少年一定會想繞過。
Discord 也提到之前 rollout 有人找到繞法。

但產品上更痛的通常是 **成人 false negative**（你是大人，但系統不承認）：

- 注重隱私的人拒絕驗證
- 光線 /鏡頭品質 /臉部年齡估計偏誤
- 共用裝置
- 新帳號、低 activity 的人

這些人不是在「做壞事」。
他們只是懶、怕麻煩、或是不想交資料。

平台可以忍受有人偷偷鑽進成人區。
比較難忍受的是：一群成年人開始覺得「我被當成嫌疑犯」。

## 3) 「裝置端」不等於「沒有隱私問題」

即使臉部估計真的在本地跑，平台還是得紀錄結果與狀態：

- 你被判定是 adult 或 teen
- 你是否可以進入限制內容
- 你是否申訴

這些都會變成新的敏感 metadata。
而且它會被接到 enforcement 邏輯裡。

如果你做過 permission system，你就知道：

- 你可以說「我們不讀私訊內容」
- 但系統仍然要建模「誰可以跟誰互動」「哪些東西能顯示」

一旦這種東西存在，它就會被使用。
不一定是惡意，常常只是因為「資料就在那」。

## 4) 更難的問題：法律要你走更硬的版本怎麼辦？

今天 Discord 嘗試把它做得像是「少數人需要」的功能。
但年齡驗證的推力很多是法律，不是文化。

如果監管要求更強的 assurance，產品自然會往這邊漂：

```text
成人內容 -> 需要強驗證
強驗證 -> 身分證件或生物特徵管線
```

可怕的不是 Discord 想要你的證件。
可怕的是 **平台把身分檢查正常化**，然後其他人跟著抄。

如果你是靠線上社群吃飯的人，你應該把它當成「先例」，而不是 Discord 的單一事件。

## 5) 工程角度：邊界越清楚，漏洞越容易長在例外上

報導裡提到未驗證用戶看到 age-restricted server 會被「黑屏遮住」。
這代表 Discord 已經在產品上劃了一個很硬的邊界：

- verified adult -> 全部看得到
- unverified -> 受限

邊界本身很簡單。
真麻煩通常會出現在例外路徑：

- 快取的 preview
- 邀請連結
- bots / integrations
- mobile / desktop 的功能差
- 申訴與帳號救援

安全事故跟隱私事故，大多不是出在主流程。
都是出在「喔對了，這個 case 也要通」的那一堆分支。

## 我自己的結論

我不羨慕任何要做這個的人。
兒少安全是真議題，法規壓力也是真的。

但如果你問我：這件事最重要的點是什麼？
不是臉掃。

是新的預設假設：

> 進入「成人空間」變成一件你要持續證明的事。

而這種模式一旦讓使用者習慣了，就很難收回去。
