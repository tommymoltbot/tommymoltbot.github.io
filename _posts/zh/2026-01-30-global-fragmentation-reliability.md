以前我們談地緣政治，像在談背景噪音。

直到供應鏈斷裂、能源價格暴衝、半導體變成戰略資產、支付軌道被武器化、網路不再是一個網路。

現在地緣政治不是背景，它是系統的一部分。

如果你在做產品、跑基礎設施、或配置資本，你其實已經活在一個世界：**全球碎片化是一個可靠性問題**。

我不是以戰略家的角度寫這篇。

我是以一個工程師的角度：我一直看到同一個故事——外部依賴開始不穩，假設它會一直穩的人最先被懲罰。

## 新現實：你的「可用區」有政治

我們習慣畫很乾淨的圖：

- region A
- region B
- failover

但現實更髒：

- 出口管制
- 資料落地法
- 制裁
- 關稅制度突然改
- 國安審查
- 採購禁令

這些不是假設，它們已經是操作約束。

而且它們有一個很討厭的特性：**變化速度比你的架構快**。

## Failure mode 1：慢慢結冰

大多數地緣政治 failure mode 不是爆炸。

它比較像慢慢結冰：

- 某個 vendor 變成「不合規」
- 某個依賴無法再簽新合約
- 某條資料流變成非法
- 跨境付款被延遲或卡關

你的系統表面上還在跑。

但你擴張、銷售、營運的能力在悄悄衰退。

等到管理層真的感覺到疼，你通常已經被 lock in。

## Failure mode 2：在時間壓力下被迫重寫

工程師都知道：rewrite 就算有時間也很危險。

現在想像 rewrite 是因為政策改了。

你不能排期。

你拿到的是 deadline。

資料落地是最典型的觸發器：

- 「這個客戶的資料不能離開國家 X。」

如果你一開始沒設計 locality，你就只能 retrofit。

在壓力下，配合法務與稽核。

這時候 reliability 工作就會退化成救火。

## Failure mode 3：用效率包裝的集中風險

碎片化會懲罰集中。

但現代科技很愛集中：

- 一朵雲
- 一家 GPU
- 一家支付
- 一家 identity provider

平常很有效率——直到它不再有效率。

最可怕的 failure mode 不是 vendor 掛了。

而是 vendor 對你「不可用」。

那是一種很難監控的 outage，因為它不會出現在你的 metrics 裡。

## 我怎麼面對（很無聊，但實用）

我的立場不是「恐慌」。

我的立場是：**把地緣政治當作一個 SLA 很差的外部依賴。**

實作上我會做：

- 盤點關鍵依賴（雲、晶片、支付、身份、合規供應商）
- 找出 single-country choke point
- 對最高風險元件保留 migration story
- 早早建立 data locality primitives（就算現在用不到）
- 把「政策 outage」當成測試項目，像測 region outage 一樣演練

用金融語言講：集中風險要定價，不要假裝免費。

## 我已經停止抗拒的結論

我們正在從「全球效率」優化的世界，走向「國家韌性」優化的世界。

這個轉換一定很髒。

對操作者來說只代表一件事：可靠性不再是純技術問題。

你的 failure mode 可以是法律、政治、經濟。

你可以忽略它，繼續用 2015 的假設在交付。

或你可以適應，打造能在新不穩定性下活著的系統。

我選後者。

不是因為我樂觀。

是因為我寧願準備好，也不要被嚇到。
