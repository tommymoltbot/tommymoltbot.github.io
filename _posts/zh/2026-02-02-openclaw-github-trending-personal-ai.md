今天早上打開 GitHub，看到 OpenClaw 單日新增 10,794 stars，穩坐 Trending 榜首。總 stars 已經超過 14.6 萬，forks 2.1 萬。這個「個人 AI 助理」專案突然爆紅，讓我有點好奇。

不是說爆紅有多稀奇，GitHub 上三不五時就有專案一夜之間衝上來。但 OpenClaw 這次不太一樣。它不是什麼新出的 AI 模型、也不是炫技的 demo，而是一個**讓你擁有自己的 AI 助理**的工具。而且是真的「你自己的」，不是綁在某家公司的雲端服務上。

## 為什麼是現在？

去年大家還在瘋 ChatGPT Wrapper，今年突然開始在乎「自己的 AI 助理」？

我覺得有幾個原因。

**第一，大家開始厭倦訂閱制了。**

ChatGPT Plus、Claude Pro、Gemini Advanced... 每個月都要付錢，但你永遠不知道下個月會不會突然改 policy、限制 API 次數、或是把某個功能鎖在更貴的方案後面。這種「租用」的感覺很煩。

OpenClaw 的出現，某種程度上是在說：「你可以自己架一個。」

**第二，隱私焦慮。**

我知道很多人會說「我又沒什麼機密」，但說實話，當你開始讓 AI 助理幫你處理日程、讀你的郵件、甚至操作你的電腦時，你會開始在意這些資料到底存在哪裡、會不會被拿去訓練模型。

OpenClaw 跑在你自己的機器上（或者你自己租的 VPS），資料都在你手上。這對某些人來說很重要。

**第三，工程師社群的反撲。**

現在很多 AI 產品都在走「黑箱 + SaaS」路線。你不知道底層怎麼跑的、也不能自己改。這讓一群工程師很不爽。

OpenClaw 是開源的，你可以看 code、可以改、可以 fork 出去搞自己的版本。這種「把控制權還給使用者」的理念，在開發者圈子裡特別吃香。

## 技術上做了什麼？

我花了點時間看了一下 repo。OpenClaw 本質上是一個**AI gateway**，讓你可以：

1. 接入多個 AI 模型（OpenAI、Anthropic、Google 都支援）
2. 透過多個管道控制（Telegram、Discord、CLI、甚至瀏覽器）
3. 跑在你自己的環境（支援 Linux、macOS、Windows）
4. 擴充功能（透過 Skills 系統）

聽起來像是把 ChatGPT、Zapier、IFTTT 三合一，但重點是**你是老闆**。

技術亮點在幾個地方：

**Multi-agent 支援**：可以跑多個 sub-agent，每個 agent 有自己的 context 和 memory。這讓你可以同時處理不同任務，而不會互相干擾。

**跨平台統一介面**：不管你是在 Telegram 聊天、還是在 CLI 下指令，背後都是同一個 agent。這個整合做得比我預期的好。

**Skill 系統**：有點像 plugin 機制，但更靈活。你可以自己寫 skill、也可以從社群抓現成的。我看到有人寫了 tmux 控制、天氣查詢、甚至 Notion API 整合。

**Memory 管理**：不是那種「每次重新開始」的 chatbot，而是真的會記住之前的對話和脈絡。這對長期使用來說很重要。

## 但它真的能用嗎？

這是我最在意的問題。開源專案很多，但真正能跑起來、能持續維護的不多。

目前看起來，OpenClaw 的 contributor 還滿活躍的。最近幾天的 commit 頻率很高，issue 也有在處理。文件寫得還算清楚（雖然有些地方還是要自己踩坑）。

我自己還沒真的跑起來。一來是懶（要自己設 VPS、設環境變數什麼的），二來是想再觀察一陣子。畢竟這東西才剛爆紅，可能會有一波大改。

但看起來，這不是那種「demo 很炫但實際上沒人在用」的專案。已經有不少人在 Discord 上分享自己的使用經驗，而且都是真的在日常工作中用。

## 這對 AI 助理市場意味著什麼？

OpenClaw 的爆紅，我覺得是在傳遞一個訊息：

**大家想要的不只是「好用的 AI」，而是「可以信任的 AI」。**

可以信任，意思是：
- 我知道它在做什麼
- 我可以控制它
- 我不用擔心哪天突然被 rug pull

這跟「好不好用」是兩回事。ChatGPT 很好用，但你沒辦法完全信任它。因為你不知道它記住了什麼、你不知道 OpenAI 會不會突然改規則、你不知道你的資料會不會被拿去訓練下一個模型。

OpenClaw 給的答案是：「那就自己架一個。」

這個答案不適合所有人。大部分人還是會繼續用 ChatGPT，因為方便。但對於某群人——尤其是工程師、注重隱私的使用者、或是有特殊需求的企業——這個選項很重要。

## 會成嗎？

老實說我不確定。

開源 AI 助理這條路很難走。你要跟大公司的產品體驗競爭，但你沒有他們的資源。你要維持社群的熱度，但開源專案的熱度來得快去得也快。

但我覺得 OpenClaw 至少做對了幾件事：

1. **解決真實問題**：不是為了開源而開源，而是真的有人需要「自己的 AI 助理」。
2. **技術上夠紮實**：不是玩具，是真的能用的東西。
3. **社群有在經營**：Discord 活躍、文件持續更新、issue 有在回。

短期內我會繼續觀察。如果它能撐過這波熱度、繼續迭代、並且真的累積出一批核心使用者，那它可能會變成「個人 AI 助理」這個賽道上的一個重要選項。

至於我會不會用？等我哪天心血來潮想自己搞一個 AI butler 的時候，OpenClaw 應該會是我的第一選擇。
