Every few months the internet produces a project that’s basically a stress test for my definition of “programming language”.

This week’s entry: **an x86 CPU emulator implemented in CSS**. As in: Cascading Style Sheets. No JS (the author says there’s an optional script tag just for a clock / stability), but the core trick is still the same: you can get a browser to evaluate *a ridiculous amount* of state transitions using pure styling machinery.

It’s one of those things that’s obviously not practical… and yet it’s not useless either, because it forces you to think about what the platform *really* is.

## Why this is interesting (beyond "haha CSS")

### 1) Browsers are a weird compute substrate

If you’ve ever shipped a big CSS codebase, you already know the dirty secret:

- the layout engine is doing work that looks suspiciously like computation
- a small change can blow up to “why is the CPU pegged at 100%?”

This project just leans into that reality and turns it into art.

### 2) The demo is a map of what the platform allows

When people say “X is Turing complete”, half the time it’s just trivia.

Here it’s more concrete:

- container queries, animations, and selector mechanics become the “clock”
- styles become your state storage
- rendering becomes your observable output

That’s not how we *want* to build software, but it is how browsers *can* be coerced.

### 3) It’s a reminder: constraints make people inventive

I don’t want anyone writing production logic in CSS.

But I do like seeing these projects because they’re proof that:

- if the platform has any side-channel for state + transitions
- someone will build a computer out of it

And the sad part is: it often teaches you more about the platform than the official docs do.

## If you’re an engineer, what do you take away?

A few boring but useful takeaways:

- “It’s just declarative” doesn’t mean “it can’t compute.”
- performance cliffs often happen where “declarative” hides a massive evaluation engine.
- security boundaries matter more than you think: once you allow a complex evaluation model, people will find weird ways to route around your assumptions.

Mostly though: it’s fun. Computers are allowed to be fun sometimes.
